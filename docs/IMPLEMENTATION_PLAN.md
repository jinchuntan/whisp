# Persephone — Implementation Plan

> Written before implementation per the build brief. Records scope, phase order,
> and the decisions that require (or explicitly avoid needing) credentials.

## 1. Goal

Persephone is an anonymous voice Q&A system for conferences. An ESP32-S3 badge records
a spoken question while a button is held, uploads a WAV over Wi-Fi, and a backend
transcribes it (Faster-Whisper locally, or Agora Real-Time STT). Similar questions
are clustered by meaning so a host can see what the room actually thinks.

## 2. Three runtime boundaries

| Boundary | Runs on | Responsibilities | Heavy deps? |
|----------|---------|------------------|-------------|
| **Web/control plane** (`persephone_api/`, `main.py`, `public/`) | Vercel (serverless FastAPI) + CDN static | HTTP API, audio ingest to storage, job creation, polling, admin dashboard | **No** — FastAPI + Supabase client only |
| **Persistent storage** (`supabase/`) | Supabase | Postgres state + private `persephone-audio` bucket | n/a |
| **Transcription worker** (`worker/`) | WSL2 Ubuntu (later a Linux container) | Claim jobs, download audio, transcribe, cluster, heartbeat | **Yes** — faster-whisper, sentence-transformers, (optional) Agora SDK |

The worker only makes **outbound** connections (to Supabase + provider APIs), so it
needs no inbound exposure. Badges talk to the Vercel API over **HTTP polling** — no
MQTT, no WebSockets.

## 3. Key design decisions (see ARCHITECTURE.md for rationale)

1. **Async upload-and-poll contract.** `POST /api/v1/questions` stores audio, inserts
   a `queued` row, and returns `202` with a `question_id` + `poll_url` immediately.
   No transcription happens inside the request (Vercel has no long tasks / background
   threads). The badge polls `GET /api/v1/questions/{id}`.
2. **State in Postgres, audio in private Storage.** Nothing persists on Vercel's
   ephemeral filesystem. Questions survive process restarts.
3. **Atomic job claim** via a Postgres function (`claim_next_question`) using
   `FOR UPDATE SKIP LOCKED`, invoked through `supabase.rpc(...)`. Expired leases are
   reclaimable so a crashed worker doesn't strand a job.
4. **Provider router** derives an ordered provider list purely from `TRANSCRIPTION_MODE`.
   Business logic never branches on provider name. Four modes: `agora_first`,
   `faster_whisper_first`, `agora_only`, `faster_whisper_only` (default).
   Providers are lazily constructed, so `faster_whisper_only` never imports, builds,
   or contacts Agora.
5. **Everything testable without credentials.** DB and storage sit behind thin
   gateways that are faked in tests. Providers are injected. Heavy ML imports are
   lazy, so the test suite runs without faster-whisper / sentence-transformers /
   Agora installed and never downloads a model or touches the network.
6. **Clustering without a paid LLM.** Local `all-MiniLM-L6-v2` embeddings + cosine
   similarity, threshold `0.78`. Cluster embeddings stored as `double precision[]`
   (no pgvector requirement); pgvector documented as an optional upgrade.

## 4. Credential inventory

| Secret | Used by | Needed to run core flow? | Notes |
|--------|---------|--------------------------|-------|
| `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` | API + worker | **Yes** (real end-to-end) | Free Supabase project. Tests fake this. |
| `BADGE_API_KEY`, `ADMIN_API_KEY` | API + firmware/dashboard | **Yes** | Generated by the user; prototype shared keys. |
| `AGORA_APP_ID`, `AGORA_APP_CERTIFICATE`, `AGORA_CUSTOMER_ID`, `AGORA_CUSTOMER_SECRET` | worker (Agora provider only) | **No** | Only consumed when mode selects Agora. Faster-Whisper path needs none. |
| Wi-Fi SSID/password | firmware | for hardware demo | Placeholders only; never committed. |

**A developer with zero Agora credentials can run the full badge→transcript→cluster
flow using Faster-Whisper.** That is the default configuration.

## 5. Agora reality check

Agora RT-STT is **not** a "POST a WAV" endpoint. It is an RTC-channel product: you
start a task via REST (v7.x: `POST /api/speech-to-text/v1/projects/{appId}/join`,
HTTP Basic auth with Customer ID/Secret), and Agora's bots join your channel and
subscribe to audio. To transcribe our pre-recorded WAV we would have to **push PCM
frames into the channel** using the Linux-only `agora-python-server-sdk` (media
bridge), then collect the transcript from the channel data-stream or a WebVTT file.

Per the brief, we do **not** fabricate a working media bridge. We ship:
- the **verified REST control-plane** (`join`/`query`/`leave`, Basic auth, token hooks);
- a **clearly defined media-bridge boundary** (`MediaBridge` protocol + verified PCM
  format: 16-bit LE mono, 16 kHz, 10 ms/320-byte frames);
- a **deterministic `MockAgoraProvider`** for tests and credential-free demos;
- a documented **installation blocker** (Linux/glibc-only SDK, real credentials +
  credit required, and unverified v7 request-body field spellings).

Everything else stays fully functional on Faster-Whisper. See `docs/AGORA_SETUP.md`.

## 6. Phase order (as delivered)

- **Phase 1** — repo skeleton, config, SQL migration, Pydantic schemas, auth, `/health`.
- **Phase 2** — audio upload, storage gateway, queued question creation, polling
  endpoint, admin state endpoint.
- **Phase 3** — Faster-Whisper worker: safe job claim, attempt recording, heartbeat,
  local end-to-end.
- **Phase 4** — dashboard: events/rounds, live feed, answered status, worker/provider
  indicators.
- **Phase 5** — semantic clustering + `similar_count`.
- **Phase 6** — ESP32 firmware (async upload+poll contract, proven pins preserved).
- **Phase 7** — Agora provider boundary, all four modes, credit-safe tests.
- **Phase 8** — Vercel config, deployment docs, final lint/type/test pass.

## 7. Testing strategy

`pytest` throughout. Tests must not: consume Agora credit, contact real Agora,
require real Supabase, download a Whisper model, or need the ESP32. Achieved via
dependency injection + fakes (`FakeDatabase`, `FakeStorage`, fake providers, fake
embedder). An optional, clearly-marked integration path exists for manual runs with
real Supabase + Faster-Whisper. `ruff` for lint/format, `mypy` where practical.

## 8. Acceptance-criteria mapping

Each numbered acceptance criterion in the brief maps to a concrete artifact; the
mapping is tracked in `README.md` ("What works / what needs setup") and the tests in
`tests/` and `worker/tests/`.
